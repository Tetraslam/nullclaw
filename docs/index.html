NullClaw docs

Product documentation.

Task-oriented guides with real commands and verified versions. The llms.txt index points agents to the same canonical pages.

Start

  • Install Get the NullClaw binary via Homebrew, Docker, or a source build with Zig 0.16.0.
  • Quickstart From a fresh install to a first conversation and a running gateway.
  • Your first channel Connect NullClaw to Telegram (or any of the 26 channels) and lock it down with allowlists.

Configure

  • Configuration One JSON file at ~/.nullclaw/config.json — OpenClaw-compatible, snake_case, no magic.
  • Providers Connect ~60 model providers, set the default route, and add fallbacks and auto-routing.
  • Memory Hybrid search over SQLite by default, nine other engines when you need them, and CLI tools to keep it clean.

Operate

  • Gateway and service Run NullClaw long-term — the HTTP gateway, pairing, cron endpoints, and background service mode.
  • Security The layers NullClaw enforces by default, the settings that widen them, and how to audit for leaked secrets.

Reference

  • CLI reference Every nullclaw command, grouped by task — setup, runtime, channels, cron, memory, automation flags, and build flags.
